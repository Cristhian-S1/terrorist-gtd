## DW schema (star schema)

Fact table: `fact_gtd_event`. 13 dimensions: `dim_ataque`, `dim_detalle`, `dim_tiempo`, `dim_lugar` (SCD Type 2 ‚Äî only one), `dim_espec_lugar`, `dim_arma`, `dim_objetivos`, `dim_gperp`, `dim_bt_grupo` (bridge table), `dim_mcr`, `dim_perpetradores`, `dim_impacto`, `dim_detalles_a`. Full DDL: `databases/scheme_2024.sql`.

## Operational gotchas (learned the hard way)

- **No `sqlplus` in this CLI environment.** Run SQL via VSCode Oracle extension, SQL Developer, or directly from Hop. The 4 SQL files in `databases/` must be executed in this order:
  1. `create_tables.sql` ‚Äî DDL (creates 25 tables + 2 sequences; drops them first)
  2. Hop pipelines load data from Excel ‚Üí bigtable ‚Üí staging ‚Üí dimensions ‚Üí fact
  3. `dim_tiempo.sql` ‚Äî PL/SQL procedure populates date dimension (run manually after DIM_TIEMPO pipeline)
  4. `post_etl_constraints.sql` ‚Äî RELY PKs, RELY DISABLE NOVALIDATE FKs, bitmap indexes on fact FKs
  5. `verification_queries.sql` ‚Äî 8 analytical test queries

- **JDBC driver is already installed** at `~/Applications/hop/lib/jdbc/ojdbc11.jar`. Do not re-download unless the driver version changes.

- **Hop metadata connection name is `inuta-gtd`** (Oracle, user `in2026_dici3`). When creating new pipelines/workflows, reuse this connection from the metadata tree ‚Äî don't recreate it per pipeline.

- **Files in `informe/` and `presentacion/` are the source of truth for pipeline design.** The `proceso_etl.md` has exact SQL, hop config, and dimension field mappings. Treat it as the blueprint, not as optional reading.

- **Design and plan docs:** `docs/superpowers/specs/2025-06-24-gtd-etl-fase2-design.md` and `docs/superpowers/plans/2025-06-24-gtd-etl-fase2-plan.md` (19 tasks).

## Conventions

- All names in Spanish (dimensions, tables, attributes, documentation).
- Dimension tables prefixed `dim_`, fact table prefixed `fact_`, staging tables suffixed `_stg`.
- Oracle ROLAP patterns from `presentacion/presentacion.md`: RELY PKs, RELY DISABLE NOVALIDATE FKs, bitmap indexes on fact FK columns, surrogate keys via `GENERATED BY DEFAULT AS IDENTITY` or explicit sequences, SCD Type 2 with `VALID_FROM`/`VALID_TO`/`VERSION` (only `dim_lugar`).
- UNPIVOT used for multi-valued columns in `dim_arma`, `dim_perpetradores`, `dim_objetivos`.
- Combination Lookup/Update for SCD Type 1; Dimension Lookup/Update for SCD Type 2.
